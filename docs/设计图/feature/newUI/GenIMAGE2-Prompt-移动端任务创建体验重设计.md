# GenIMAGE2 Prompt：移动端任务创建体验重设计

## 设计目标

- 页面 / 组件名称：TimeHello 移动端任务创建体验重设计
- 用户角色：移动端任务管理用户，常在碎片时间快速记录待办、安排提醒、用语音或 AI 生成任务
- 核心业务目标：让用户通过文字、语音或 AI 快速创建任务，并在创建前确认任务标题、时间、清单、标签、优先级
- 主要操作：输入任务、点击麦克风录音、停止录音、查看转写预览、AI 生成任务卡片、批量确认创建、打开底部设置抽屉

## 页面信息架构

- 顶部区域：日期、今日概览、菜单按钮、AI 入口
- 主内容区：今日任务列表，任务卡片包含完成圆点、标题、时间、标签、番茄数、优先级
- 语音浮层：大圆角底部面板，包含 AI 模式标识、提示文案、波形、计时、停止按钮、取消按钮
- AI 确认页：任务卡片列表，每张卡显示标题、时间、清单、标签、编辑和删除
- 底部区域：固定快速创建条，包含输入框、麦克风、AI 按钮、加号
- 底部抽屉：创建设置，包含日期、时间、清单、标签、优先级、番茄数、更多设置

## 视觉风格

- 风格关键词：清爽、移动端原生、任务管理、轻 AI、语音输入、柔和但高效
- 色彩倾向：浅暖灰背景、白色卡片、柔和粉红语音面板、红色录音停止按钮、绿色完成状态、少量蓝紫 AI 标识
- 信息密度：中等，手机一屏可以看见 3-5 条任务，控件点击区域清晰
- 圆角 / 阴影 / 间距：卡片 16px 圆角，底部面板 28px 圆角，按钮胶囊或圆形，阴影克制
- 是否偏企业后台 / 运营系统 / 数据看板 / 移动端 / C 端产品：移动端 C 端效率工具

## 交互状态

- 默认态：任务首页展示底部快速创建条
- 录音态：底部语音面板显示波形和计时
- 转写中：语音面板显示“正在转写”，停止按钮变为 loading
- 转写完成：显示转写文本预览和三个按钮“重新录制 / 插入 / 直接创建”
- AI 结果态：展示任务卡片确认列表和固定创建按钮
- 设置态：底部抽屉展示常用任务设置
- 异常态：录音权限或转写失败时显示明确提示和重试按钮

## 设计约束

- 保留 TimeHello 任务管理属性，不能像营销 landing page
- 必须新增语音入口、AI 入口、转写预览、任务卡片确认、底部设置抽屉
- 必须弱化复杂表单和日历优先视图
- 不允许把所有设置堆在输入框旁边
- 需要兼容 iPhone 尺寸，画面建议使用 390x844 或 430x932 的手机屏幕比例

## 输出要求

- 生成一张高保真移动端 UI 概念设计图
- 图中展示 4 个并排手机界面：
  1. 任务首页 + 底部快速创建条
  2. 语音录制/转写预览面板
  3. AI 生成任务卡片确认页
  4. 创建设置底部抽屉
- 中文 UI 文案清晰可读
- 布局适合 Flutter 前端开发还原
- 不要生成网页落地页，不要生成桌面后台，不要使用夸张渐变或装饰性插画

## UI 文案建议

- 首页标题：今天
- 输入占位：添加任务，或说出来
- 语音标题：说出你要做的事
- 转写中：正在转写...
- 转写完成按钮：重新录制、插入、直接创建
- AI 页标题：AI 已整理 4 个任务
- 批量按钮：创建 4 个任务
- 设置抽屉标题：任务设置
- 常用项：今天、明天、本周、清单、标签、优先级、番茄数
