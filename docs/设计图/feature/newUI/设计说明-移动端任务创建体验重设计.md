# 设计说明：移动端任务创建体验重设计

## 1. 设计背景

- 需求来源：`docs/feature/newUI/需求文档-移动端任务管理体验升级.md`
- 页面 / 功能名称：移动端任务创建、语音输入、AI 任务卡片确认、创建设置抽屉
- 设计类型：现有页面改版 + 新组件设计 + 交互优化
- 设计目标：让移动端用户可以用更少步骤完成任务创建，并在语音和 AI 结果确认中获得清晰反馈。

## 4. 用户需求摘要

用户希望移动端任务管理能力更完整，尤其是创建任务时可以使用语音选择或语音输入，并参考 AppAIPlugin 的录音体验。产品侧已经确认需要让 UI 设计师重新设计移动端创建任务体验，不只是开发直接加按钮。

## 5. 设计目标

- 降低移动端创建任务的操作成本。
- 让普通创建、语音创建和 AI 创建有统一入口。
- 让语音录制、转写中、转写失败和确认插入有清晰状态。
- 让 AI 生成任务后优先以卡片确认，而不是日历优先。
- 将复杂设置从输入框附近移入底部抽屉，减少拥挤。

## 6. 页面结构

### 6.1 任务首页

- 顶部区域：日期、今日任务概览、菜单入口。
- 主内容区域：今日任务列表，任务卡片展示标题、时间、标签、番茄数和完成状态。
- 操作区域：底部固定快速创建条，包含输入框、语音按钮、AI 创建按钮和新增按钮。
- 数据展示区域：按“今天 / 稍后 / 已完成”组织任务。
- 弹窗 / 抽屉：语音录制面板、创建设置抽屉。

### 6.2 语音录制面板

- 面板形式：底部半屏或大圆角浮层。
- 核心元素：AI 模式标识、提示标题、波形、计时、停止按钮、取消按钮。
- 转写完成：展示文本预览，支持“重新录制 / 插入 / 直接创建”。
- 失败状态：展示失败原因和重试按钮。

### 6.3 AI 任务卡片确认页

- 顶部区域：AI 输入摘要和任务数量。
- 主内容区域：任务卡片列表，每张卡展示标题、时间、清单、标签、优先级。
- 操作区域：每张卡支持编辑、删除；底部固定“创建 N 个任务”。
- 辅助区域：日历预览放入折叠区或二级 Tab。

### 6.4 创建设置抽屉

- 常用设置：日期、时间、清单、标签、优先级、番茄数。
- 高级设置：提醒、重复、目标、备注、任务类型。
- 交互方式：常用项以 chip 展示，点开进入具体选择。

## 7. 交互说明

| 场景 | 交互 | 预期 |
|---|---|---|
| 快速创建 | 用户点击底部输入条输入文字或点麦克风 | 立即进入文本或语音创建流程 |
| 语音录制 | 点击麦克风后弹出录音面板并开始计时 | 用户知道当前正在录音 |
| 停止录音 | 点击红色停止按钮 | 进入转写中状态 |
| 转写完成 | 展示转写文本预览 | 用户可以检查识别结果 |
| 转写失败 | 展示失败提示和重试按钮 | 用户不会误以为任务已创建 |
| AI 生成 | 用户发送文本或语音给 AI | 返回任务卡片列表 |
| 批量确认 | 用户删除/编辑卡片后点击创建 | 只创建用户确认后的任务 |
| 设置任务 | 用户点击设置 chip 或更多设置 | 打开底部抽屉，不挤压输入框 |

## 8. 页面状态

| 状态 | 设计说明 |
|---|---|
| 默认态 | 底部创建条常驻，任务列表保持清爽，主操作明确 |
| 加载态 | AI 解析和语音转写显示进度文案，不阻塞取消 |
| 空态 | 展示示例句，如“明早 9 点提醒我交周报” |
| 异常态 | 录音权限、上传失败、转写失败分别给可操作反馈 |
| 禁用态 | AI 请求中禁用发送，语音转写中禁用重复停止 |
| 成功态 | 创建成功后以 toast 或列表新任务高亮反馈 |

## 9. 视觉规范

| 项目 | 说明 |
|---|---|
| 色彩 | 以浅暖灰背景、白色卡片、红色语音强调、绿色任务完成强调为主 |
| 字体 | 标题 18-22，任务标题 15-16，辅助信息 12-13 |
| 圆角 | 卡片 14-18，底部面板 24-32，按钮 999 圆形或胶囊 |
| 阴影 | 仅用于底部输入条和浮层，避免列表过重 |
| 信息密度 | 移动端保持中等密度，任务卡片一屏可见 3-5 条 |
| 图标 | 麦克风、AI、日历、标签、清单、番茄数使用统一线性图标 |

## 10. 组件拆分建议

- `MobileMissionQuickCreateBar`：移动端底部快速创建条。
- `MobileMissionVoiceInputPanel`：语音录制、转写和文本确认。
- `MobileAIMissionConfirmList`：AI 任务确认列表容器。
- `MobileAIMissionConfirmCard`：AI 任务单卡。
- `MobileMissionCreateSettingsSheet`：移动端创建设置抽屉。

## 11. 开发注意点

- 语音能力必须复用 `AppAiVoiceRecordUtils` 和 `VoiceTranscriptionManager`。
- 音频上传必须继续按 `DocType.audio` 处理。
- MVP 不新增 Mission 字段，所有结果最终仍落 `MissionModel`。
- 新组件需要补中文文件头和关键流程注释。
- `BottomBar` 先做移动端增量替换，不要一次性大重构桌面逻辑。

## 12. 本次设计重点

- 把“创建任务”作为移动端第一主操作。
- 把语音从隐藏能力变成显性入口。
- 把 AI 结果从文本列表变成可确认任务卡。
- 把复杂设置从输入框附近移走，变成底部抽屉。
