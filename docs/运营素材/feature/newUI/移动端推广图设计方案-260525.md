# 时间管理局 ToDo 移动端推广图设计方案

## 1. 本次目标

为 App 上线准备一套中文移动端推广图，优先用于 App Store iPhone 截图位和上线前内部审核。设计重点不是重画 App 界面，而是用真实 iPhone 截图做核心内容，在外层补充上线包装、主标题、场景标签和品牌识别。

## 2. 尺寸与平台

- 平台：iOS / App Store / iPhone 竖屏截图位
- 本次输出尺寸：`1290 x 2796`
- 选择原因：Apple App Store Connect 官方截图规格中，iPhone 6.9 英寸显示可接受 `1290 x 2796` 竖屏图；旧版 `1284 x 2778` 属于 6.5 英寸规格，可作为兼容备份。
- 官方参考：https://developer.apple.com/help/app-store-connect/reference/app-information/screenshot-specifications

## 4. 图片结构

统一结构：

1. 顶部小品牌：`时间管理局 ToDo`
2. 顶部大标题：每张只讲一个卖点
3. 副标题：解释用户收益，控制在 1 行到 2 行
4. 中部设备框：嵌入真实 App 截图，App 内部 UI 不重画、不替换
5. 底部场景标签：帮助搜索和转化，但不堆关键词
6. 底部来源说明：`真实产品截图 + 上线包装，不伪造 App 内部界面`

视觉方向：

- 背景使用浅色工作台风格，避免过重渐变和装饰。
- 每张用不同功能色做左侧识别条，形成系列感。
- 手机截图占画面主体，保证 App UI 在缩略图里仍能看清。
- 文案放在顶部，适合 App Store 搜索页和产品页快速扫读。

## 6. 推荐上架排序

1. 把任务快速记下来：第一张要让用户立刻知道这是任务管理 / ToDo 工具。
2. 四象限看清重点：强化差异化，不只是普通清单。
3. 日程安排更清楚：表达时间规划能力。
4. 移动端设置完整：补充创建页深度能力。
5. 复盘专注投入：等有真实数据后再放入最终上架位。

如果最终只能上传 4 张，建议先移除当前第 4 张数据复盘图，等重截有数据版本后再补。

## 8. 上线风险与待确认

- 当前只生成中文版本；英文版本不能直接套用中文 App UI，需要切换真实英文环境后重新截图。
- AI / 语音能力没有可用真实移动端截图，本次没有生成最终 AI 卖点图；如果要把 AI 放在前 3 张，需要先采集真实 AI / 语音页面。
- 数据复盘图当前使用空数据截图，视觉表达偏弱；最终提交前建议用 release 或 TestFlight 包造一组真实数据后重截。
- 这些图片可以用于上线前设计评审和素材草案；作为最终审核素材前，应再用当前 release 构建确认页面无 debug 红字、无权限弹窗、无真实隐私数据。
