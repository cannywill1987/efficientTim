# GenIMAGE2 Prompt - 时间管理局 ToDo 移动端推广图

## 1. 本次状态

本次已通过本地脚本生成中文移动端推广图：

- `docs/运营素材/feature/newUI/appstore-mobile-promo-v2/zh/`

因为 App Store 上架截图必须保留真实 App UI，本轮没有让图片生成模型重画 App 内部页面。下面 Prompt 用于后续生成“外层背景 / 设备包装 / 运营氛围参考”，生成后仍需要用本地脚本把真实截图嵌入，不能直接把 AI 生成的 App UI 当作上架截图。

## 2. Prompt

```text
请生成一套 App Store 移动端推广图的外层包装视觉参考，不要生成或重画 App 内部界面。

产品：时间管理局 ToDo
类型：时间管理、待办清单、四象限任务、日程规划、番茄专注
语言：中文
尺寸：1290 x 2796 px，iPhone 竖屏 App Store 截图比例

硬性要求：
1. App 内部 UI 必须来自后续嵌入的真实截图，不得在图中生成假的 App 页面。
2. 画面需要预留一个居中的 iPhone 设备框区域，用于后续放入真实截图。
3. 只生成外层背景、品牌区域、标题排版、卖点标签、轻量装饰和设备外壳风格参考。
4. 不要让文字和手机区域重叠；顶部标题需要足够醒目，适合 App Store 搜索页缩略图。
5. 风格清爽、现代、效率工具感，避免过度炫酷、过多装饰和夸张营销。

系列结构：
第一张：把任务快速记下来
副标题：标题、标签、优先级与时间设置，一屏完成日常任务创建。

第二张：四象限看清重点
副标题：按重要与紧急程度拆分任务，先处理真正关键的事情。

第三张：日程安排更清楚
副标题：月历和任务列表结合，计划每天的节奏与空档。

第四张：复盘专注投入
副标题：任务数、专注时长与完成率可视化，看见效率变化。

第五张：移动端设置完整
副标题：提醒、清单、日期和番茄数都能随手配置，减少来回跳转。

视觉约束：
- 每张保持同一版式系统。
- 每张只使用一个主功能色作为识别，不要变成单一大面积纯色背景。
- 背景以浅色、干净、轻量纸面感为主。
- 底部可放 2 到 3 个场景标签，例如：任务管理、四象限、日程规划、专注时长。
- 保留“真实产品截图 + 上线包装，不伪造 App 内部界面”的安全边界。
```

## 3. 后处理要求

生成外层包装参考后，必须执行本地合成：

1. 把真实 App 截图嵌入设备框。
2. 检查 App UI 没有被 AI 改写。
3. 输出 `1290 x 2796` PNG。
4. 上传前确认每张图无 debug 红字、无权限弹窗、无真实隐私数据。
