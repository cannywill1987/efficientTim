# 页面如何跳转

## 目标

这份文档用于约束项目里的页面跳转方式，避免出现：

- 桌面端页面切了，但左侧菜单高亮没跟上
- 页面跳转直接 `Navigator.push`，结果绕开了项目自己的路由状态
- 主内容区、任务详情区、右侧详情区混用同一套跳转方法，导致页面错位

如果任务涉及页面跳转、桌面端菜单切换、`Env` 路由状态、或新增 PC 页面入口，请先看这份文档。

## 一、先分清 3 套跳转

### 1. 主内容区切页

适用场景：

- 左侧菜单点击后切换主页面
- 例如 `Skills`、`社区`、`设置`

要用：

```dart
Utility.pushDesktopMainContainerNavigator(context, 'SkillHomePage', {});
```

原因：

- 这一套会写入 `Env.routerMainContainerData`
- 左侧菜单高亮就是根据这个状态判断的
- `DesktopRouter.getMainPage()` 也是根据它来切主页面

### 2. 中间区域内部跳转

适用场景：

- 从列表进入详情
- 从文件夹页进入任务页

参考 [FoldersPage.dart](/Users/linzhibin/Desktop/code/MangaKingFlutter/lib/com/timehello/page/folderspage/FoldersPage.dart)：

```dart
Utility.pushDesktopNavigator(context, 'MissionPage', {'data': data});
```

原因：

- 这一套会写入 `Env.routerData`
- 用于中间内容区内部的局部切换
- 不会改变左侧主菜单高亮

### 3. 普通 Flutter 页面 push

适用场景：

- 弹独立子页
- 临时详情页
- 不需要参与 PC 主菜单状态的页面

例如：

```dart
Navigator.of(context).push(
  MaterialPageRoute<void>(
    builder: (BuildContext context) => const SkillFavoritePage(),
  ),
);
```

不要把这套直接拿来做桌面主菜单切页。

## 二、桌面端新增页面的标准步骤

如果要新增一个能被左侧菜单切换的页面，必须同时改 4 处。

### 1. 新建页面文件

建议页面参考 `BaseWidget` 写法，例如：

- [SkillHomePage.dart](/Users/linzhibin/Desktop/code/MangaKingFlutter/lib/com/timehello/page/SkillHomePage/SkillHomePage.dart)

### 2. 在菜单配置里加入口

文件：

- [CONSTANTS.dart](/Users/linzhibin/Desktop/code/MangaKingFlutter/lib/com/timehello/config/CONSTANTS.dart)

方法：

- `CONSTANTS.getLeftDesktopMenubar(...)`

每个菜单项至少要有：

- `sceneCode`
- `icon`
- `iconChecked`
- `title`

### 3. 在 `DesktopRouter` 里接页面

文件：

- [DesktopRouter.dart](/Users/linzhibin/Desktop/code/MangaKingFlutter/lib/com/timehello/page/DesktopRouter.dart)

需要同时改：

- `buildBodyFunction(...)` 里的 `Offstage`
- `reportEvent(...)` 埋点
- 如果默认首页要变，也要改这里的默认 `page`

### 4. 用主内容区跳转方法切过去

菜单点击后要走：

```dart
Utility.pushDesktopMainContainerNavigator(context, sceneCode, {});
```

不能直接 `Navigator.push`。

## 三、为什么 `FoldersPage` 是好参考

[FoldersPage.dart](/Users/linzhibin/Desktop/code/MangaKingFlutter/lib/com/timehello/page/folderspage/FoldersPage.dart) 里面有两个很重要的示例：

### 1. 手机端和桌面端分流

```dart
if (Utility.isHandsetBySize()) {
  Utility.pushNavigator(context, MissionPage(mangaModel: data));
} else {
  Utility.pushDesktopNavigator(context, 'MissionPage', {'data': data});
}
```

这说明：

- 手机端更适合普通 `Navigator`
- 桌面端内部区域切换要优先走项目封装路由

### 2. 页面本身不要直接硬管菜单状态

`FoldersPage` 只负责触发跳转，不直接改左侧菜单组件。  
菜单高亮由 `Env.routerMainContainerData` 和 `PCLeftMenuBarWidget` 统一控制。

## 四、什么时候该改 `routerMainContainerData`

应该改：

- 左侧菜单切页
- 主导航切页
- 默认首页切换

不应该改：

- Skill 卡片详情弹窗
- 项目管理弹窗
- 普通对话框
- 只在当前页面内部展示的详情区域

## 五、什么时候该改 `routerData`

应该改：

- 番茄钟主内容内部的二级页面
- 任务详情
- 列表到详情

参考：

```dart
Utility.pushDesktopNavigator(context, 'MissionPage', {'data': data});
```

## 六、最容易犯错的地方

### 1. 只加了菜单，没接路由

表现：

- 左侧点了有高亮
- 右边页面不切换

### 2. 只接了路由，没加菜单

表现：

- 页面代码存在
- 用户找不到入口

### 3. 默认首页只改了一处

默认首页如果要从 `TomatoPage` 改成 `SkillHomePage`，至少要检查：

- `DesktopRouter.buildBodyFunction(...)`
- `PCLeftMenuBarWidget` 的默认高亮 fallback
- `CONSTANTS.getLeftDesktopMenubar(...)` 的菜单顺序

### 4. 主内容切页误用 `pushDesktopNavigator`

表现：

- 页面看起来切了
- 但左侧菜单高亮还是旧的

原因：

- `pushDesktopNavigator` 改的是 `routerData`
- 不是 `routerMainContainerData`

## 七、推荐检查清单

做页面跳转相关改动后，至少确认：

- 左侧菜单显示顺序是否正确
- 默认首页是否正确
- 点击菜单后主内容是否切换
- 左侧菜单高亮是否同步
- `flutter analyze` 是否通过

## 八、一句话经验

桌面端主菜单切页看 `routerMainContainerData`，  
桌面端内容区内部跳页看 `routerData`，  
普通独立子页才走 `Navigator.push`。
