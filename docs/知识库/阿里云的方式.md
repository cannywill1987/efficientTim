# 阿里云 OSS 上传方式

## 适用场景

当需求明确涉及阿里云 OSS 直传、上传音频、上传录音文件、上传大文件，或保存业务数据前需要先拿到 OSS URL 时，优先参考本文。

如果只是普通图片或普通文件上传，也需要先确认是否可以复用 [oss上传方法.md](/Users/linzhibin/Desktop/work/project/flutter/efficientTimeFinal/efficientTime5/efficientTime/docs/知识库/oss上传方法.md) 里的 `HttpManager.uploadImage(...)` / `HttpManager.uploadFile(...)`。只有确认要走客户端直传 OSS 时，才使用本文的 `AliyunStoreManager` 链路。

## 核心封装

Flutter 客户端的阿里云 OSS 直传封装在：

- [AliyunStoreManager.dart](/Users/linzhibin/Desktop/work/project/flutter/efficientTimeFinal/efficientTime5/efficientTime/lib/com/timehello/util/AliyunStoreManager.dart)

常用方法：

```dart
String url = await AliyunStoreManager.getInstance().uploadFileByFilePath(
  docType: DocType.audio,
  path: file.path,
  fileName: Utility.getUUID(),
);
```

## 推荐参数

- `path`：本地文件路径，通常来自录音、选文件、临时目录或缓存目录。
- `docType`：上传文件类型。录音必须传 `DocType.audio`，避免落到默认 `image` 目录。
- `fileName`：推荐传 `Utility.getUUID()`，不要直接复用用户输入，避免特殊字符、重名和隐私信息进入 OSS key。
- `downloadCallback`：需要进度 UI 时再传，用于接收上传进度。

## 返回值

`uploadFileByFilePath(...)` 成功时返回最终 OSS URL，例如：

```dart
https://.../timehello/audio/{uid}/{uuid}{random}.jpg
```

注意：当前方法默认 `fileExtensionEnum` 是 `FileExtension.jpg`。如果业务强依赖真实扩展名，需要先检查 `FileExtension` 枚举和播放端兼容性，再决定是否传入更准确的扩展名。

## 保存业务数据

OSS 上传只负责生成文件 URL，不负责写业务数据。

典型顺序：

1. 先生成或拿到本地文件。
2. 调用 `AliyunStoreManager.getInstance().uploadFileByFilePath(...)` 上传到 OSS。
3. 拿到返回 URL 后，再写入对应业务模型的 `extra`、`noteRecordUrls`、`wqb...RecordUrls` 或其他字段。
4. 同时保存本地路径、文件大小、时长等辅助信息，方便离线、回放和调试。

示例：

```dart
File file = File(path);
String url = await AliyunStoreManager.getInstance().uploadFileByFilePath(
  docType: DocType.audio,
  path: file.path,
  fileName: Utility.getUUID(),
);

final extra = jsonEncode({
  "file": file.lengthSync(),
  "duration": duration,
  "url": url,
  "localUrl": file.path,
});
```

## 注意事项

- 不要在页面里重新手写 OSS client、STS 获取、bucket 路径拼接。
- 不要把上传逻辑散落在多个组件里，优先封装到页面的保存方法或业务 service。
- 音频上传必须显式传 `DocType.audio`。
- 上传前要确认文件存在，上传后要检查 URL 非空，再继续写业务数据。
- 如果页面有 `Confirm` / `保存` 交互，推荐在确认时上传和保存，不要录完立即保存，避免用户还没预览就生成脏数据。
