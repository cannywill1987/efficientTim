## 修复记录

### UI-001 番茄钟首页统计卡溢出

- 现象：`0小时25分钟`、图标和标签在 4 列指标中横向溢出，Debug 模式显示红色 `RIGHT OVERFLOWED BY ... PIXELS`。
- 原因：每个指标只有约 73px 宽，原实现用 `Row` 同时承载图标、数字、单位，内容天然宽度超过列宽。
- 修改：将移动端指标数值行改为 `Text.rich + WidgetSpan` 内联渲染，避免子 `Row` 先触发受限布局溢出；数字字体轻微收敛。
- 验证：`tomato-list-mobile-after.png` 中红色 overflow 标记已消失。
- before：`screenshots/tomato-list-mobile-before.png`
- after：`screenshots/tomato-list-mobile-after.png`

### UI-002 我的页宫格底部溢出

- 现象：功能宫格每个 item 出现 `BOTTOM OVERFLOWED BY 24/41 PIXELS`，上架截图不可用。
- 原因：四列宫格 `childAspectRatio: 0.98` 让 item 高度接近宽度，但 item 内实际包含 54px 图标卡、双行标题和副标题，内容高度超过格子高度。
- 修改：将宫格 `childAspectRatio` 调整为 `0.62`，并把主轴间距调整为 18，让单个入口有足够高度。
- 验证：`profile-mobile-after.png` 中红色 overflow 标记已消失，标题/副标题完整展示。
- before：`screenshots/profile-mobile-before.png`
- after：`screenshots/profile-mobile-after.png`

### UI-003 底部导航留白过高

- 现象：底部胶囊导航下方到 iOS Home Indicator 之间留白过高，视觉上形成一大块空白。
- 原因：我的页滚动内容额外保留了 `120` 底部 padding，同时公共底部胶囊导航自身有 `82` 高度和 `16` 底部 margin，再叠加 SafeArea。
- 修改：我的页底部内容 padding 收敛到 `24`；公共底部胶囊导航高度从 `82` 调整到 `74`，底部 margin 从 `16` 调整到 `6`。
- 验证：`profile-mobile-after-compact-bottom.png` 中底部空白明显降低，仍保留 Home Indicator 安全距离。
- before：用户反馈截图、`screenshots/profile-mobile-after.png`
- after：`screenshots/profile-mobile-after-compact-bottom.png`

### UI-009 我的页首屏遮挡与个人卡溢出

- 现象：登录态“我的”页绿色个人卡内部出现黑黄 overflow 标记；功能宫格第三行标题和副标题被底部悬浮导航压住，首屏可读性不足。
- 原因：iOS 移动端仍在个人卡底部渲染客户端下载入口，导致卡片内部高度超出；资料卡高度偏高，和底部悬浮导航叠加后压缩了宫格可见区域。
- 修改：iOS/Mac 端不在个人卡内展示客户端下载入口；专注时长文案限制 2 行并收敛字号；登录态资料卡高度从 `238` 收敛到 `210`；滚动内容保留底部导航安全距离。
- 验证：`profile-mobile-after-safe-scroll.png` 中黑黄 overflow 标记消失，第三行宫格标题和副标题已露出，底部导航不再压住主要文字。
- before：用户反馈截图、`screenshots/profile-mobile-before.png`
- after：`screenshots/profile-mobile-after-safe-scroll.png`

### UI-010 我的页个人卡信息层级

- 现象：绿色个人卡虽然不再溢出，但头像、用户名、排名、专注时间、时薪、金币和“我要花”按钮没有稳定层级，右侧收益信息被缩得过小，整体像被压扁。
- 原因：原实现直接复用 `MoneyHandlerWidget` 和旧资料区布局，组件天然尺寸不适合塞进移动端绿色卡片；中间身份信息和右侧收益信息互相抢宽。
- 修改：资料卡改为左头像、中身份、右收益三栏；用户名、排名、专注时长独立展示；右侧收益改为移动端专用轻量展示，避免复用宽组件导致整体缩放；头像和卡片高度同步收敛。
- 验证：`profile-mobile-after-profile-card-redesign.png` 中个人卡不再出现黑黄 overflow，排名/专注时间/金币按固定区域展示，功能宫格第三行保持可读。
- before：用户反馈截图、`screenshots/profile-mobile-after-safe-scroll.png`
- after：`screenshots/profile-mobile-after-profile-card-redesign.png`

### UI-005 侧边清单抽屉可读性

- 现象：抽屉左右留白过大，顶部标题只能显示成 `时间管理局To...`；清单行文字列过窄，长标题容易不自然换行；底部固定升级卡压缩列表高度，影响继续浏览“清单/过滤器/标签/归档”等入口。
- 原因：新版移动端抽屉宽度仅占屏幕 `0.79`，同时列表项左右 margin 为 `28`；头部番茄图标、关闭按钮和间距偏大；升级卡固定放在抽屉底部，不参与列表滚动。
- 修改：抽屉宽度调整为 `0.88`；头部图标、关闭按钮和间距收敛；列表项左右 margin 收敛到 `14`，内部 padding 收敛到 `8`；清单标题允许最多 2 行；右侧统计列重新留出宽度；移动端抽屉移除固定升级卡，优先保障清单导航可见性。
- 验证：`side-drawer-mobile-after-readable.png` 中顶部标题完整，系统清单和右侧统计值不再异常换行；`side-drawer-mobile-after-lower.png` 中下方入口可以继续浏览。
- before：`screenshots/side-drawer-mobile-before.png`、用户反馈截图
- after：`screenshots/side-drawer-mobile-after-readable.png`、`screenshots/side-drawer-mobile-after-lower.png`

### UI-008 番茄钟列表可读性与边距

- 现象：顶部统计区数值字号偏大，任务列表卡片左右边距过宽，长任务标题、日期区在窄屏下容易被截断，和旧版移动端列表的自然换行表现不一致。
- 原因：新版移动端样式仍沿用偏大的统计字号和宽外边距；任务标题 `maxLines` 原先只给 1 行，卡片内部横向间距压缩了文本可用宽度。
- 修改：收敛顶部统计数值、单位和标签字号；统计值按列宽轻微自适应缩放，优先完整显示时间文本；列表外边距从 `26` 收敛到 `12`；任务卡片左右内边距和操作区间距收窄；任务标题允许最多 2 行展示，日期/评分区继续使用 `Wrap` 承接换行。
- 验证：`tomato-list-mobile-after-readable.png` 中顶部统计无溢出，`0小时25分钟`、`0小时00分钟` 完整展示；列表卡片更接近旧版宽度，短标题正常展示；长标题会进入 2 行而不是过早截断。
- before：用户反馈截图、`screenshots/tomato-list-mobile-before.png`
- after：`screenshots/tomato-list-mobile-after-readable.png`

### UI-006 创建任务移动端创建效率

- 现象：创建任务页首屏只有返回、关闭和右上角创建按钮，缺少页面标题；语音创建没有在任务创建主流程露出；底部创建按钮在长表单末尾，移动端用户需要滚动到底部才有明确提交入口。
- 基线对比：当前真实截图 `screenshots/create-mission-mobile-before.png` 显示首屏信息密度高，设置项从首屏底部开始露出，底部提交不稳定；用户前序需求希望移动端也有任务管理和语音创建能力。
- 原因：创建页仍以完整设置表单为主体，没有为移动端高频“快速创建”建立固定底部操作和语音入口；标题区只依赖右侧按钮，页面目的不够明确。
- 修改：AppBar 增加居中标题“创建任务”；标题区下方增加“录制语音”快捷卡片，复用 `RecordPage2` 的录音、预览、上传和转写链路，确认后把转写文本回填到标题/备注；移动端提交按钮固定到底部安全区上方，桌面端仍保留原来的表单底部按钮。
- 验证：`create-mission-mobile-after.png` 中标题、语音入口和固定创建按钮均可见；向下滚动时固定按钮不阻断设置项浏览，底部仍可继续滑动查看备注和重复设置。
- before：`screenshots/create-mission-mobile-before.png`
- reference/concept：无，本次直接基于真实截图和用户语音创建需求落地
- after：`screenshots/create-mission-mobile-after.png`

### UI-011 创建任务移动端视觉重构

- 现象：第一版移动端创建页虽然补齐了语音入口和固定创建按钮，但整体仍偏表单化；顶部蓝灰背景、四象限旧按钮、设置区散列表和参考图的轻卡片风格不一致。

- 修改：移动端创建页改为暖白画布；AppBar 收敛成圆角返回/关闭/创建按钮；标题、添加标签、录制语音、四象限、选择壁纸、设置列表全部调整为参考图里的轻卡片结构；四象限标题使用自适应缩放，避免“不紧急 & 重要”等文案被截断；设置列表复用 `MenuItem2` 新增的创建任务移动端样式。
- 验证：`create-mission-mobile-after-genimage-style.png` 中首屏结构已经贴近参考图，四象限标题完整展示，设置卡片和底部创建按钮可见。

### UI-012 创建任务底部固定按钮遮挡

- 现象：移动端创建页底部固定蓝色“创建”按钮遮挡设置列表，首屏只能露出“保存模式”一行，影响继续配置开始时间、完成时间、清单等字段。

- 修改：隐藏移动端底部固定提交条，仅保留顶部右侧“创建”；滚动区底部 padding 从安全区 + `92` 收敛到安全区 + `24`。
- 验证：`create-mission-mobile-after-hide-bottom-submit.png` 中底部按钮已消失，设置列表自然露出到“开始时间”。
- before：用户反馈截图、`screenshots/create-mission-mobile-after-genimage-style.png`
